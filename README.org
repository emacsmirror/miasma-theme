Emacs color theme inspired by the woods. It is mostly a direct port of [[https://github.com/xero/miasma.nvim][Miasma]] theme.

[[https://raw.githubusercontent.com/daut/miasma-theme.el/main/screenshots/preview.png]]

** Installation
=miasma-theme= is available on MELPA. Ensure MELPA is added to your package archives, and then add the following to your =init.el= file.
#+begin_src emacs-lisp
(use-package miasma-theme
  :config
  (load-theme 'miasma t))
#+end_src
*** Manual
Download =miasma-theme.el= file to the =~/.emacs.d/themes/= directory. Add these lines to your =init.el= file.
#+begin_src emacs-lisp
(add-to-list 'custom-theme-load-path (concat user-emacs-directory "/themes/"))
(load-theme 'miasma t)
#+end_src

** Configuration

*** LSP Mode

By default, Miasma preserves syntax highlighting colors for "unnecessary" code (such as unused imports or variables) detected by LSP mode. Instead of overriding the meaningful colors that indicate variables, functions, and other language elements, only a wavy underline is displayed.

This behavior can be customized by setting ~lsp-diagnostics-attributes~ in your configuration *after* loading the theme:

#+begin_src emacs-lisp
(load-theme 'miasma t)

;; Example: Apply gray foreground to unnecessary code
(setq lsp-diagnostics-attributes
      '((unnecessary :foreground "gray")
        (deprecated :strike-through t)))
#+end_src

If you have customized ~lsp-diagnostics-attributes~ via ~M-x customize-variable~, your preference will be automatically respected.
